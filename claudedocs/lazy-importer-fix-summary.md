# Lazy Importer Bug Fix Summary

**Date**: 2026-01-25
**Status**: Core bug fixed, circular import issues remain

## Problems Found

### 1. ✅ FIXED: Incorrect Import Module Path
**Issue**: Line 179 of `lazy_importer.py` was calling:
```python
module: ModuleType = import_module(module_name, package=package)
```

With `module_name="metadata"` and `package="codeweaver.core"`, Python was trying to import a top-level `metadata` module instead of `codeweaver.core.metadata`.

**Fix**: Changed to relative import:
```python
module: ModuleType = import_module(f".{module_name}", package=package)
```

### 2. ✅ FIXED: Python 3.13 NamedTuple Compatibility
**Issue**: Python 3.13 doesn't allow explicit `__match_args__` definition in NamedTuple.

**File**: `src/codeweaver/core/spans.py`

**Fix**: Removed explicit `__match_args__` definition (line 95):
```python
# Before:
__match_args__: ClassVar[tuple[str, str]] = ("start", "end")

# After: (comment only)
# Note: __match_args__ is automatically generated by NamedTuple
```

### 3. ✅ FIXED: Python 3.13 Pydantic Private Attributes
**Issue**: Pydantic doesn't allow field names with leading underscores in Python 3.13.

**File**: `src/codeweaver/core/chunks.py`

**Fix**: Changed `_embeddings` from `Field()` to `PrivateAttr()`:
```python
# Before:
_embeddings: dict[str, BatchKeys] = Field(
    default_factory=dict,
    description="Mapping of embedding intents to their corresponding batch keys.",
    serialization_alias="embeddings",
    exclude=True,
)

# After:
from pydantic import PrivateAttr
_embeddings: dict[str, BatchKeys] = PrivateAttr(default_factory=dict)
```

### 4. ✅ FIXED: Circular Import in core/dependencies.py
**Issue**: `core/dependencies.py` imported `TelemetryService` at module level, creating circular dependency with `core/telemetry/client.py`.

**File**: `src/codeweaver/core/dependencies.py`

**Fix**: Moved import to TYPE_CHECKING and used forward references:
```python
# Before:
from codeweaver.core import TelemetryService

async def _create_telemetry_service(...) -> TelemetryService:
type TelemetryServiceDep = Annotated[TelemetryService, ...]

# After:
# Import removed from top level

async def _create_telemetry_service(...) -> "TelemetryService":
    from codeweaver.core.telemetry.client import TelemetryService  # Local import
type TelemetryServiceDep = Annotated["TelemetryService", ...]  # Forward reference
```

### 5. ✅ FIXED: Deep Circular Import Chain
**Issue**: Complex circular dependency chain between core/dependencies.py and providers modules.

**Root Cause**: Multiple issues creating circular dependencies:
1. `core/dependencies.py` lines 92-110 used `importlib.util.find_spec()` to conditionally import settings classes at module initialization time
2. Runtime imports of types needed only for type annotations
3. Pydantic incompatibilities (Field with both default and default_factory)
4. TypedDict qualifiers (NotRequired, Required, total=False) used on Pydantic models

**Fixes Applied**:
1. Moved conditional settings imports to TYPE_CHECKING block, used Union of forward reference strings
2. Moved runtime imports (SessionStatistics, DictView, etc.) out of TYPE_CHECKING
3. Added missing lazy import mapping for SummaryKey in core/types/__init__.py
4. Fixed lazy import mapping for AllDefaultProviderSettings (config.providers not providers)
5. Moved EmbeddingCapabilityGroupDep import to TYPE_CHECKING, used forward reference
6. Fixed Pydantic compatibility issues:
   - Removed `= None` from fields with `default_factory` in SessionStatistics
   - Removed `total=False` from CollectionConfig (BaseModel, not TypedDict)
   - Removed `NotRequired` from FastEmbedProviderMixin fields (added `= None` defaults)
   - Removed `Required` from AgentProviderSettings fields (not needed for BaseModel)
7. Fixed import from wrong package (SparseEmbedding from core not providers)
8. Fixed pydantic.dataclasses.as_dict to standard library dataclasses.asdict
9. Added HAS_FASTEMBED check locally instead of importing from test code
