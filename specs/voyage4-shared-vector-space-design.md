# Voyage-4 Shared Vector Space Support Design

## Executive Summary

This document specifies the architectural changes required to support **Voyage-4's cross-model querying capability**, where different models in the Voyage-4 family (voyage-4-large, voyage-4, voyage-4-lite, voyage-4-nano) can query embeddings generated by each other because they share the same vector space.

### Key Challenge

Current CodeWeaver architecture assumes:
1. ✗ Query and embed use the **same model**
2. ✗ Mismatched model names = **error condition**
3. ✗ Query and embed use the **same provider** (e.g., both VOYAGE or both SENTENCE_TRANSFORMERS)

Voyage-4's innovation breaks all three assumptions:
- ✓ Can query with `voyage-4-nano` (local, SENTENCE_TRANSFORMERS provider)
- ✓ Against embeddings from `voyage-4-large` (API, VOYAGE provider)
- ✓ Achieves **3-point improvement** in retrieval quality vs. single-model approaches

---

## Problem Analysis

### Current System Constraints

#### 1. **Single-Model Assumption in Providers**
**Location**: `src/codeweaver/providers/embedding/providers/base.py`

```python
@property
def model_name(self) -> ModelNameT:
    """Get the model name for the embedding provider."""
    if self.caps:
        return ModelName(self.caps.name)
    return ModelName(self.config.model_name or ...)
```

- Each provider instance = one model
- Both `embed_documents()` and `embed_query()` use same model
- No support for asymmetric model pairs

#### 2. **Model Name Validation in Vector Store**
**Location**: `src/codeweaver/providers/vector_stores/metadata.py`

```python
def validate_compatibility(self, other: CollectionMetadata) -> None:
    if self.dense_model and other.dense_model and \
       self.dense_model != other.dense_model:
        raise ModelSwitchError(
            f"Your existing embedding collection was created with model "
            f"'{other.dense_model}', but the current model is '{self.dense_model}'."
        )
```

- Blocks model switching at collection level
- No concept of "model family" or "compatible models"
- String equality check prevents cross-model operations

#### 3. **Provider Isolation in Configuration**
**Location**: `src/codeweaver/config/providers.py`

```python
class EmbeddingProviderSettings(BaseProviderCategorySettings):
    model_name: ModelNameT
    embedding_config: EmbeddingConfigT  # Single provider config
```

- One provider config per operation (embed vs. query)
- No mechanism to specify "embed with VOYAGE, query with SENTENCE_TRANSFORMERS"

#### 4. **No Query-Time Model Validation**
**Location**: `src/codeweaver/agent_api/find_code/pipeline.py`

- Query embedding generated without checking compatibility with index
- Dimension mismatch only detected at vector store search phase
- No early warning about incompatible model families

## Proposed Solution

### Design Principles

1. **Constitutional Compliance**: Follow "Proven Patterns" principle - extend existing architecture rather than reinvent
2. **Evidence-Based**: Solution must be testable and verifiable
3. **Backward Compatibility**: Existing single-model usage must work unchanged
4. **Progressive Enhancement**: Opt-in for advanced features (shared vector space)

### Architecture Changes

#### **Phase 1: Model Family Abstraction**

##### 1.1 Create `ModelFamily` Capability Metadata

**Location**: `src/codeweaver/providers/embedding/capabilities/base.py`

```python
from typing import Literal, FrozenSet
from pydantic import Field

class ModelFamily(BasedModel):
    """Defines a group of models that share compatible vector spaces."""

    family_id: str = Field(
        description="Unique identifier for the model family (e.g., 'voyage-4')"
    )

    vector_space_dimension: PositiveInt = Field(
        description="Shared dimension across all family members"
    )

    vector_space_datatype: str = Field(
        description="Shared datatype (float32, uint8, etc.)"
    )

    is_normalized: bool = Field(
        description="Whether vectors are L2-normalized"
    )

    preferred_metrics: tuple[str, ...] = Field(
        description="Distance metrics suitable for this family"
    )

    member_models: FrozenSet[str] = Field(
        description="Set of model names in this family"
    )

    asymmetric_query_models: FrozenSet[str] | None = Field(
        default=None,
        description="Subset of models optimized for query (if different from embed)"
    )

    cross_provider_compatible: bool = Field(
        default=False,
        description="Whether family members can span multiple providers"
    )

    def is_compatible(self, embed_model: str, query_model: str) -> bool:
        """Check if two models can be used together."""
        return (
            embed_model in self.member_models and
            query_model in self.member_models
        )

    def validate_dimensions(
        self,
        embed_dim: int,
        query_dim: int
    ) -> tuple[bool, str | None]:
        """Validate that dimensions match family expectations."""
        if embed_dim != self.vector_space_dimension:
            return False, f"Embed dimension {embed_dim} != family dimension {self.vector_space_dimension}"
        if query_dim != self.vector_space_dimension:
            return False, f"Query dimension {query_dim} != family dimension {self.vector_space_dimension}"
        return True, None


class EmbeddingModelCapabilities(BasedModel):
    # ... existing fields ...

    model_family: ModelFamily | None = Field(
        default=None,
        description="Optional model family for cross-model compatibility"
    )
```

#### **Phase 2: Asymmetric Model Configuration**

##### 2.1 Extend Configuration Schema

**Location**: `src/codeweaver/config/providers.py`

```python
class AsymmetricEmbeddingConfig(BasedModel):
    """Configuration for using different models for embed and query."""

    embed_provider_settings: EmbeddingProviderSettings = Field(
        description="Provider settings for document embedding"
    )

    query_provider_settings: EmbeddingProviderSettings = Field(
        description="Provider settings for query embedding"
    )

    validate_family_compatibility: bool = Field(
        default=True,
        description="Whether to validate models belong to same family"
    )

    @model_validator(mode="after")
    def validate_model_family_compatibility(self) -> Self:
        """Ensure embed and query models are compatible."""
        if not self.validate_family_compatibility:
            return self

        embed_caps = _get_embedding_capabilities_for_model(
            self.embed_provider_settings.model_name
        )
        query_caps = _get_embedding_capabilities_for_model(
            self.query_provider_settings.model_name
        )

        if not embed_caps or not query_caps:
            raise ValueError("Model capabilities not found for one or both models")

        # Check if both have model families
        if not (embed_caps.model_family and query_caps.model_family):
            raise ValueError(
                "Asymmetric configuration requires both models to belong to a model family"
            )

        # Verify same family
        if embed_caps.model_family.family_id != query_caps.model_family.family_id:
            raise ValueError(
                f"Model family mismatch: embed uses '{embed_caps.model_family.family_id}', "
                f"query uses '{query_caps.model_family.family_id}'"
            )

        # Validate specific model pairing
        family = embed_caps.model_family
        if not family.is_compatible(
            str(self.embed_provider_settings.model_name),
            str(self.query_provider_settings.model_name)
        ):
            raise ValueError(
                f"Models '{self.embed_provider_settings.model_name}' and "
                f"'{self.query_provider_settings.model_name}' are not compatible within family"
            )

        return self
```

##### 2.2 Update Settings to Support Asymmetric Mode

**Location**: `src/codeweaver/config/settings.py`

```python
class CodeWeaverSettings(BaseSettings):
    # Existing embedding settings (backward compatible)
    embedding: EmbeddingProviderSettings | None = None
    sparse_embedding: SparseEmbeddingProviderSettings | None = None

    # New asymmetric embedding mode
    asymmetric_embedding: AsymmetricEmbeddingConfig | None = Field(
        default=None,
        description="Optional asymmetric embedding configuration (different models for embed/query)"
    )

    @model_validator(mode="after")
    def validate_embedding_config(self) -> Self:
        """Ensure only one embedding configuration mode is active."""
        symmetric = self.embedding is not None
        asymmetric = self.asymmetric_embedding is not None

        if symmetric and asymmetric:
            raise ValueError(
                "Cannot specify both 'embedding' and 'asymmetric_embedding'. "
                "Use 'asymmetric_embedding' for different embed/query models, "
                "or 'embedding' for traditional single-model configuration."
            )

        if not symmetric and not asymmetric:
            # Apply default embedding config
            self.embedding = _get_default_embedding_settings()

        return self
```

#### **Phase 3: Vector Store Compatibility Updates**

##### 3.1 Extend Collection Metadata

**Location**: `src/codeweaver/providers/vector_stores/metadata.py`

```python
class CollectionMetadata(BasedModel):
    # ... existing fields ...

    dense_model_family: str | None = Field(
        default=None,
        description="Model family ID if using cross-model compatibility"
    )

    query_model: str | None = Field(
        default=None,
        description="Query model if different from embedding model (asymmetric mode)"
    )

    def validate_compatibility(self, other: "CollectionMetadata") -> None:
        """Validate collection metadata against provider configuration."""

        # Provider switch detection (warning only)
        if self.provider != other.provider:
            logger.warning(
                "Provider switch detected: collection created with '%s', "
                "but current provider is '%s'.",
                other.provider,
                self.provider,
            )

        # NEW: Model family compatibility check
        if self.dense_model_family and other.dense_model_family:
            if self.dense_model_family == other.dense_model_family:
                # Same family - validate specific model compatibility
                family = _resolve_model_family(self.dense_model_family)
                if family:
                    current_query_model = self.query_model or self.dense_model
                    stored_query_model = other.query_model or other.dense_model

                    if not family.is_compatible(
                        str(self.dense_model),
                        str(current_query_model)
                    ):
                        logger.warning(
                            "Query model '%s' may not be optimal for embeddings "
                            "from '%s' in family '%s'",
                            current_query_model,
                            self.dense_model,
                            self.dense_model_family,
                        )
                else:
                    logger.warning(
                        "Could not resolve model family '%s' for compatibility validation",
                        self.dense_model_family,
                    )
            else:
                # Different families - strict incompatibility
                raise ModelSwitchError(
                    f"Model family mismatch: collection uses family '{other.dense_model_family}', "
                    f"but current configuration uses family '{self.dense_model_family}'.",
                    details={
                        "stored_family": other.dense_model_family,
                        "current_family": self.dense_model_family,
                        "stored_model": other.dense_model,
                        "current_model": self.dense_model,
                    },
                    suggestions=[
                        "Reindex with the new model family",
                        f"Revert to model family '{other.dense_model_family}'",
                        "Delete and recreate collection: cw index --force --clear",
                        "Create a new collection with a different name",
                    ],
                )

        # MODIFIED: Traditional model switch check (only if no family defined)
        elif self.dense_model and other.dense_model and \
             self.dense_model != other.dense_model:
            # Traditional strict model matching for non-family models
            raise ModelSwitchError(
                f"Your existing embedding collection was created with model "
                f"'{other.dense_model}', but the current model is '{self.dense_model}'. "
                f"You can't use different embedding models for the same collection.",
                details={
                    "stored_model": other.dense_model,
                    "current_model": self.dense_model,
                    "query_model": self.query_model,
                },
                suggestions=[
                    "Reindex with the new embedding model",
                    f"Revert to model '{other.dense_model}'",
                    "Delete and recreate collection: cw index --force --clear",
                    "Create a new collection with a different name",
                ],
            )
```

##### 3.2 Update Base Vector Store

**Location**: `src/codeweaver/providers/vector_stores/base.py`

```python
class VectorStoreProvider[StoreClient](BasedModel, ABC):
    # ... existing fields ...

    @property
    def query_model(self) -> ModelNameT | None:
        """Get the query model (may differ from dense_model in asymmetric mode)."""
        # Check if asymmetric config is being used
        if hasattr(self, "_query_capabilities") and self._query_capabilities:
            return (
                self._query_capabilities.config.model_name
                if hasattr(self._query_capabilities, "config")
                else None
            )
        # Fallback to dense model for backward compatibility
        return self.dense_model

    def _create_collection_metadata(self) -> CollectionMetadata:
        """Create metadata for new collection."""
        dense_caps = self.embedding_capabilities.dense

        return CollectionMetadata(
            provider=self.provider.variable,
            created_at=datetime.now(tz=timezone.utc),
            project_name=self.project_name,
            dense_model=str(dense_caps.config.model_name) if dense_caps else None,
            sparse_model=self.sparse_model,
            backup_enabled=False,
            backup_model=None,
            collection_name=self.collection_name,
            version="1.3.0",  # Bump version for family support
            dense_model_family=(
                dense_caps.capability.model_family.family_id
                if dense_caps and dense_caps.capability.model_family
                else None
            ),
            query_model=(
                str(self.query_model)
                if self.query_model and self.query_model != self.dense_model
                else None
            ),
        )
```

## Conclusion

This design provides a **constitutional-compliant** (proven patterns, evidence-based) approach to supporting Voyage-4's shared vector space while maintaining backward compatibility. The asymmetric embedding configuration is opt-in, allowing users to benefit from 3-point quality improvements and cost reductions without disrupting existing deployments.

**Key Benefits**:
- ✅ 3-point retrieval quality improvement
- ✅ Zero query cost (local inference)
- ✅ Instant query latency (no API calls)
